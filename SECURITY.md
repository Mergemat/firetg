# Security policy

## Supported versions

Security fixes are provided for the latest published version of firetg.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub private
vulnerability reporting from the repository's **Security** tab. If that option
is unavailable, contact the maintainer through the private contact method on
their GitHub profile.

Include the affected version, impact, reproduction steps, and any suggested
mitigation. Do not include live Telegram API credentials, session files,
authentication codes, passwords, or private message content.
